# Output Contract

Every skill emits artifacts into `runs/<slug>/` following a stable shape so subagents can read each other's work without guesswork. All three schemas below are **authoritative** — skills that don't conform must be fixed, not worked around.

`<slug>` format: `YYYY-MM-DD_<topic-kebab>`, generated by `orchestrate-multi-ai` at the start of a run.

## 1. Raw source answer — `runs/<slug>/raw/<source>.json`

One file per AI site or tool, written by a `browser-operator` or `research-scout` invocation. Even on failure you MUST write this file, with `error` populated.

```json
{
  "source": "gpt",
  "model": "gpt-4o",
  "question": "<the exact prompt sent to the site>",
  "answer_md": "<markdown body of the assistant reply, or null on error>",
  "url": "https://chatgpt.com/c/<conversation-id>",
  "ts": "2026-04-13T12:34:56Z",
  "citations": [
    { "title": "...", "url": "...", "snippet": "..." }
  ],
  "error": null
}
```

- `source` — stable short name: `gpt`, `gemini`, `notebooklm`, `grok`, `x-scan`, `tavily-search`, etc.
- `citations` — optional array; empty array is fine, but include it when the site surfaces sources (NotebookLM, Gemini, Grok).
- On failure, set `answer_md: null` and `error: { stage, message, screenshot_path? }`.

Tavily outputs live in their own subfolder: `runs/<slug>/raw/tavily/search.json`, `raw/tavily/extract/<domain>.json`, `raw/tavily/crawl/<domain>.json`. Each wraps the raw Tavily response under a `result` key and adds the same `source` / `ts` / `error` fields.

## 2. Evidence pack — `runs/<slug>/evidence.json`

Produced by `evidence-verifier` (via the `build-evidence-pack` skill). Normalizes the raw files into atomic claims with provenance.

```json
{
  "slug": "2026-04-13_moe-architectures",
  "question": "<user's original question>",
  "created": "2026-04-13T12:40:00Z",
  "claims": [
    {
      "id": "c01",
      "text": "<single-sentence factual claim>",
      "sources": ["gpt", "gemini", "tavily:arxiv.org/abs/..."],
      "confidence": "high|medium|low",
      "notes": "<optional reviewer note>"
    }
  ],
  "contradictions": [
    { "claim_ids": ["c03", "c07"], "note": "<how they disagree>" }
  ],
  "open_questions": [
    "<things no source addressed>"
  ]
}
```

## 3. Final brief — `runs/<slug>/brief.md`

Produced by `synthesis-editor`. Reads `evidence.json` only (not raw). Max 800 words unless the user asked otherwise. Frontmatter is required and must match exactly so Obsidian search and Dataview queries keep working.

```markdown
---
title: <Human Title>
slug: 2026-04-13_moe-architectures
created: 2026-04-13T12:45:00Z
tags: [teamx, <topic-tag>]
sources: [gpt, gemini, notebooklm, grok, tavily]
question: <user's original question>
---

## TL;DR
<2–3 sentences>

## Key findings
- ...

## Disagreements
- ...

## Open questions
- ...

## Sources
- [gpt] runs/<slug>/raw/gpt.json
- [gemini] runs/<slug>/raw/gemini.json
- ...
```

## Failure discipline

A skill that can't produce its artifact MUST still write a stub file with `error` populated so the downstream verifier can see it tried. Silent skips break the evidence chain.
